## 离散化技巧

**lower_bound** 用在stl中 查找第一个“大于或等于”目标值的元素位置。

因为返回的是迭代器，我们通常需要把它转换成整数下标。 **公式：** `迭代器 - 起始迭代器 = 下标`

```c++
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<int> a(n); // 原始数组
    set<int> st;      // 辅助set，用于自动排序和去重
    for (int i = 0; i < n; i++) {
        cin >> a[i];
        st.insert(a[i]); // 插入set，自动变有序且无重复
    }
    // --- 关键步骤 ---
    // set 不支持快速查找下标，所以我们需要把 set 里的东西倒到一个 vector 里
    vector<int> b(st.begin(), st.end()); 
    for (int i = 0; i < n; i++) {
        // lower_bound 在 b 中查找第一个 >= a[i] 的位置
        // 减去 b.begin() 得到的就是下标（即排名，从0开始）
        int rank = lower_bound(b.begin(), b.end(), a[i]) - b.begin();
        cout << rank;
        if(i!=n-1) cout<<" ";
    }
    return 0;
}
```

### 最小涂色操作数

也是模拟 找好规律 别怕这种题

A. 颜色的本质拆解
题目中说：

红色（R） = 横着涂（行操作）

蓝色（B） = 竖着涂（列操作）

绿色（G） = 横着涂 + 竖着涂（既要红又要蓝）

关键洞察： 这就意味着，我们可以把问题完全割裂开看：

只看红色需求（包括 R 和 G）：这一层只需要考虑行操作。

只看蓝色需求（包括 B 和 G）：这一层只需要考虑列操作。

最后的答案 = 红色所需的行操作数 + 蓝色所需的列操作数。

因为行操作和列操作互不干扰（除了在 G 处重叠，但重叠正是我们想要的），所以可以分开计算。

B. 转化为“数线段”问题
既然拆开了，问题就变成了：

在每一行里，有多少段连续的红色区域？每一段都需要涂一次。

在每一列里，有多少段连续的蓝色区域？每一段都需要涂一次。

怎么数连续区域？ 这是经典的贪心策略：只有当遇到一个新的红色块，且它的前一个不是红色块时，我们才认为需要新起一笔。 如果前一个已经是红色，那我们可以顺势把这一笔延长，不需要额外操作。

```c++
#include<bits/stdc++.h>
using namespace std;

int main() {
    int H, W;
    cin >> H >> W;
    vector<string> grid(H);
    for (auto &row : grid) {
        cin >> row;
    }
    
    int ans = 0;
    // 统计需要的最小行操作数
    for (int i = 0; i < H; ++i) {
        for (int j = 0; j < W; ++j) {
            if (grid[i][j] == 'R' || grid[i][j] == 'G') {
                if (j == 0 || !(grid[i][j - 1] == 'R' || grid[i][j - 1] == 'G')) {
                    ++ans;                                                        // 增加行操作计数
                }
            }
        }
    }
    
    // 统计需要的最小列操作数
    for (int j = 0; j < W; ++j) {
        for (int i = 0; i < H; ++i) {
            if (grid[i][j] == 'B' || grid[i][j] == 'G') {
                if (i == 0 || !(grid[i - 1][j] == 'B' || grid[i - 1][j] == 'G')) {
                    ++ans;                                                        // 增加列操作计数
                }
            }
        }
    }
    
    cout << ans << '\n';
    return 0;
}
```

### 前缀和 基础赛-3 找出分界点 

在快速排序算法中，分区操作是一个关键步骤。该操作会从数组中选择一个基准元素（称为 pivot），然后将数组重新排列，使得所有小于等于基准的元素都位于基准的左侧，所有大于基准的元素都位于基准的右侧。这个基准元素最终会被放置在它在最终有序数组中应该出现的位置上。

现在给定一个已经完成分区操作的数组，请判断哪些元素有可能作为该分区操作的基准元素。请找出所有满足上述条件的元素，按照它们在原数组中出现的顺序输出。

```c++
#include<bits/stdc++.h>
using namespace std;
int main(){
    int n;cin>>n;
    vector<int> v;
    vector<int> ans;
    for(int i=0;i<n;i++){
        int tmp;cin>>tmp;
        v.push_back(tmp);
    }
    // 他前面所有的都比他小，他后面所有的都比他大
   for (int i = 0; i < n; i++) {
    int a = v[i];
    bool is_valid = true; // 假设它是合格的
    // 检查前面：必须都比它小
    for (int j = 0; j < i; j++) {
        if (v[j] >= a) { // 发现反例
            is_valid = false;
            break; // 这里的 break 跳出 j 循环
        }
    }
    // 如果前面已经不合格了，就不需要检查后面了，直接下一轮 i
    if (!is_valid) continue; // 这里的 continue 跳过本次 i，进入 i++

    // 检查后面：必须都比它大
    for (int k = i + 1; k < n; k++) {
        if (v[k] <= a) { // 发现反例（注意这里通常是 v[k] < a，具体看题目是否允许相等）
            is_valid = false;
            break; // 这里的 break 跳出 k 循环
        }
    }

    // 只有当 is_valid 依然为 true 时，才记录答案
    if (is_valid) ans.push_back(a);
       
	}
    int sz = ans.size();
    cout<<sz<<endl;
    for(int i=0;i<sz;i++){
        cout<<ans[i];
        if(i!=sz-1)cout<<" ";
    }
    return 0;
}
```



一种朴素的做法是，对于每个位置 i，我们向左扫描检查所有元素是否都小于 ai，再向右扫描检查所有元素是否都大于ai，这样总的时间复杂度会达到 On^2。会超时。



前缀和——空间换时间，通过一次 O(N)的预处理，让后续所有的“区间求和”查询都能在O(1)的时间内完成

1. 为什么需要前缀和？

假设你有一个数组 $A$，有很多次询问：

- 第 1 次问：计算第 2 到第 5 个数的和。
- 第 2 次问：计算第 1 到第 1000 个数的和。
- ...
- 第 M 次问：计算第 L 到第 R 个数的和。

暴力做法：

每次询问都写一个 for 循环从 $L$ 加到 $R$。

- 如果有 $M$ 次询问，最坏时间复杂度是 $O(M \times N)$。如果数据量大（例如 $N=10^5, M=10^5$），程序会直接超时 (Time Limit Exceeded)。

前缀和做法：**“预处理”**思想（类似前缀和），将 O(N^2)$的暴力解法优化到了 O(N)。

预处理 O(N)，每次查询 O(1)。总复杂度 O(N + M)。飞快！



### 2-SUM-hash

用空间换时间

2-SUM-hash

![image-20260110205813035](C:\Users\86173\AppData\Roaming\Typora\typora-user-images\image-20260110205813035.png)

**核心逻辑：从“寻找”变为“查表”**

假设你要在操场上的一堆人里，找到两个人，他们的体重加起来正好等于 $K$ 斤。

1. 笨办法（暴力法 $O(N^2)$）

你抓起第一个人 A，然后跑遍全操场去问剩下的每个人：“嘿，你体重是 $K - A$ 吗？”

问完一圈，再抓起第二个人 B，再去跑遍全操场问……

效率： 累死人。如果人多，根本跑不完。

2. Hash 思想（查表法 $O(N)$）—— 也就是图片里的解法

为了不跑断腿，你拿来一本**花名册（Hash 数组/布尔数组）**。

第一步：登记（预处理）

所有人进场时，你只做一件事：问他的体重。

比如来了个体重 3 的人，你就在花名册第 3 页打个勾（hashTable[3] = true）。

这一步只要遍历一遍所有人。

第二步：配对（查表）

现在你拿着花名册，站在队伍前，看着每一个人。

假设目标总和 $K = 10$。

- 你看到一个人，体重是 **3**。
- 你心里想：如果要配对成功，他的搭档必须是 **7**（因为 $10 - 3 = 7$）。
- **关键动作：** 你不需要去操场找人，你只需要**低头看一眼手里的花名册**，第 7 页有没有打钩？
  - 如果有钩 $\rightarrow$ 找到了！`ans++`
  - 如果没钩 $\rightarrow$ 没戏，下一个。

第三步：去重（除以 2）

为什么要除以 2？

因为当你遇到体重 3 的人时，你查到了 7 在场，你喊了一次“配对成功”。

等到你走到体重 7 的那个人面前时，你心里想“我要找 3”，你查表发现 3 在场，你又喊了一次“配对成功”。

同一对人（3和7），被你算了两次。所以最后答案要 ans / 2。

```c++
#include<bits/stdc++.h>
using namespace std;
const int MAXK = 1000001;
int main() {
    int n, k;
    cin >> n >> k;                                 // 读取元素个数和目标和
    vector<int> a(n);                              // 存储输入的数组
    vector<bool> hashTable(MAXK, false);           // 记录数值是否出现
    for (int i = 0; i < n; i++) {
        cin >> a[i];                               // 读取数组元素
        hashTable[a[i]] = true;                    // 标记元素出现
    }
    int ans = 0;                                   // 计数答案
    for (int x : a) {
        if (k - x >= 0 && hashTable[k - x]) ans++; // 如果存在补数，计数加一
    }
    cout << ans / 2;                               // 输出结果
    return 0;
}
```

## 注意力机制的FLOPs浮点运算次数

### 1. 为什么叫 FLOPs？

- **FLOPs** 指的是浮点运算的总数（加法+乘法）。
- 在 GPU 上，矩阵乘法是核心计算。一次“乘加运算”（Multiply-Add）通常计为 **2 FLOPs**（一次乘法，一次加法）。

### 2. 标准 Attention 的 FLOPs 是怎么算的？

这是你需要背下来的公式，面试和考试都会用到。

假设我们有：

- 序列长度 (Sequence Length): O(N)
- 隐藏层维度 (Hidden Dimension): d(或 dhead)
- 输入矩阵 Q, K, V 的形状都是 [N, d]

Attention 的核心公式是：

![image-20260112193748781](C:\Users\86173\AppData\Roaming\Typora\typora-user-images\image-20260112193748781.png)

#### **第一步：计算分数 (Score Calculation) $Q \times K^T$**

- **操作：** 矩阵 $Q$ ($N \times d$) 乘以 矩阵 $K^T$ ($d \times N$)。
- **结果形状：** $N \times N$ (这就是著名的 Attention Map)。
- **运算量：** 两个矩阵相乘，结果有 $N^2$ 个元素，每个元素需要 $d$ 次乘法和 $d$ 次加法。
- **FLOPs：** $2 \cdot N^2 \cdot d$

#### **第二步：加权求和 (Weighted Sum) $Score \times V$**

- **操作：** Attention Map ($N \times N$) 乘以 矩阵 $V$ ($N \times d$)。
- **结果形状：** $N \times d$。
- **运算量：** 结果有 $N \cdot d$ 个元素，每个元素需要 $N$ 次乘法和 $N$ 次加法。
- **FLOPs：** $2 \cdot N^2 \cdot d$

#### **结论：Attention 层的总 FLOPs**

将两步相加，核心 Attention 机制的计算量约为：



$$\text{Total FLOPs} \approx 4N^2d$$

------

### 3. 这个“4N²d”意味着什么？(CS336 的重点)

#### **1. $N^2$ 的诅咒 (Quadratic Complexity)**

你会发现公式里有 $N^2$。这意味着：

- 如果你把输入的提示词长度（Context Length）翻倍（比如从 4k 扩充到 8k），Attention 的计算量会变成原来的 **4倍**。
- 这就是为什么长文本（Long Context）在大模型中极其消耗算力和显存的原因。

#### **2. 相比于 MLP 层**

大模型里除了 Attention 还有 FFN (Feed Forward Network)。FFN 的计算量是 $O(N)$（线性的）。

- 当序列 $N$ 很短时，运算量主要在 FFN。
- 当序列 $N$ 很长时，Attention 的 $N^2$ 就会主导整个模型的计算瓶颈。

------

### 4. 另一种可能性：内存翻转 (Transpose / Permute)

虽然概率较小，如果你看到的资料真的在讲“Flip/Transpose”（转置）：

- 在实现 Multi-head Attention 时，我们需要改变 Tensor 的维度顺序。
- 例如从 `[Batch, Sequence, Heads, Dim]` 变成 `[Batch, Heads, Sequence, Dim]`。
- 在 PyTorch 中这叫 `.permute()` 或 `.transpose()`。
- **CS336 的考点：** 这种操作在内存中是不连续的，会导致缓存命中率（Cache Miss）下降。**FlashAttention** 的核心贡献之一就是通过分块计算（Tiling），减少了这种昂贵的内存读写操作。

### 总结

你复习的时候，把“翻转次数”理解为 **FLOPs（运算次数）** 即可。

记住这个结论用于应试：

Attention 的时间复杂度是 $O(N^2)$，具体的 FLOPs 数量大约是 $4N^2d$。

## 260120 “Clash Verge 系统代理下 Edge 无法联网

### 1. 故障现象 (Symptoms)

- **环境**：Windows 11, Clash Verge 系统代理(System Proxy Mode, `127.0.0.1:8196`).
- **表现**：Chrome 可以正常访问网络，但 Edge 无法打开任何网页。
- **初步排查**：关闭安全 DNS、检查代理插件、开启 IPv6、检查系统代理设置均无效。
- 

### 2. 根本原因 (Root Cause)

- **核心机制**：**Windows UWP 应用沙盒隔离 (AppContainer Sandbox)**。
- **原理**：Windows 为了安全，默认禁止 UWP应用访问本地回环接口（Loopback Interface, 即 `127.0.0.1`）。
- **冲突点**：虽然 Edge 基于 Chromium 内核，但在 Windows 上它部分遵循 UWP 规范。当 Clash 在本地 `127.0.0.1` 建立代理服务时，Edge 被系统防火墙拦截，无法与本地代理建立 TCP 连接。

### 3. 排查过程中的关键转折 (Critical Path)

这次排查成功的关键在于通过 **控制变量法** 和 **底层工具** 锁定了问题域：

1. **排除服务端问题**：使用 PowerShell 调用 `curl.exe -x ...` 成功返回 Google 源码。
   - *结论*：证明 Clash 代理服务工作正常，Socket 监听正常。问题 100% 锁定在 Edge 客户端本身。
2. **发现操作误区**：这是最坑的一点。你最初使用 "Loopback Exemption Utility" 时，**手动勾选**了一个 `Microsoft Edge` 进程，但问题未解决。
   - *原因*：Edge 是**多进程架构**（Browser Process, GPU Process, Network Service, Renderer Process）。你手动勾选的可能只是主 UI 进程，而真正负责发包的“网络服务进程”依然被关在沙盒里。

### 4. 最终解决方案 (Final Solution)

- **工具**：点开clash verge中的UWP工具![image-20260120175607467](C:\Users\86173\AppData\Roaming\Typora\typora-user-images\image-20260120175607467.png)AppContainer Loopback Exemption Utility。
- **操作**：点击 **`Exempt All` (全选解除)** 按钮，然后保存。
- **逻辑**：不纠结于具体是哪个 Edge 子进程受限，直接将系统中所有 UWP 容器的回环限制全部解除（Whitelist All），彻底打通本地流量。

### 5. 知识点映射

- **进程隔离与安全 (Process Isolation)**：操作系统如何通过沙盒机制限制进程的资源访问权限（这里是网络资源）。
- **回环接口 (Loopback Interface)**：在网络层，`127.0.0.1` 的流量是直接在协议栈内部“短路”回来的，不经过物理网卡，但依然受操作系统的网络访问控制列表 (ACL) 约束。

# 医药
