## 一、正则化（Regularization）

### 什么是正则化？

正则化是一类**防止模型过拟合**的技术的统称。

**核心思想**：故意给学习过程"增加难度/加噪声"，**逼迫模型学到更通用、更本质的规律**，而不是死记训练数据。

**类比**：一个学生如果只死记硬背考试题的答案，他在做过的原题上得分很高，但换一套新题就不行了。正则化就是逼学生理解知识，而不是背答案。

### 常见正则化手段

| 方法 | 原理 | 你实验中是否用到 |
|---|---|---|
| **L1/L2 正则化** | 在损失函数里加惩罚项，限制模型参数不能太大 | L2 通过 weight decay 间接使用 ✅ |
| **Dropout** | 训练时随机"关掉"一些神经元 | ERNIE 模型内部自带 ✅ |
| **数据增强** | 对训练数据做变换（如 token masking） | 5% token ID 掩码 ✅ |
| **Early Stopping** | 验证集性能不再提升时提前停止训练 | epoch=2 本身就很少 ✅ |
| **Weight Decay** | 每次更新时让权重乘以略小于 1 的系数 | weight_decay=0.01 ✅ |

### 面试回答模板

> 正则化是防止过拟合的一类技术的统称。我的实验中用到了多种正则化手段：AdamW 的解耦 weight decay（0.01）、5% token ID 掩码数据增强、FGM 对抗训练。其中 **token masking 类似 Dropout 的思路**，都是随机丢弃一部分信息，迫使模型不依赖某些特定特征，从而提高泛化能力。

## 二、掩码数据发生在什么阶段？

### 关键区分：预训练 vs 微调

| | BERT 预训练的 MLM | 你实验中的 Token Masking |
|---|---|---|
| **阶段** | 预训练阶段 | **微调阶段** |
| **目的** | 让模型预测被 mask 的词 → 学语言表示 | 数据增强/正则化 → 提高泛化 |
| **mask 率** | 15% | 5% |
| **mask 方式** | 替换为 `[MASK]` token | 替换为 `0`（PAD token） |
| **是否设计预测任务** | 是，有专门的 MLM loss | **否**，不预测被 mask 的词 |

### 具体发生时机

```
训练数据 → tokenizer 编码为 input_ids → ⭐ 5% token 随机替换为 0 → 送入模型 → 计算 loss
                                         ↑
                                    在这里做 masking
                                  （数据预处理阶段）
```

masking 发生在**微调阶段的数据预处理时**，在模型真正读到这条数据之前，`input_ids` 就已经被随机替换了。

### 面试回答模板

> 我的 token masking 是在**微调阶段**进行的，不是预训练阶段。具体来说，在数据预处理时，以 5% 的概率将 `input_ids` 中的 token 随机替换为 0（PAD token）。这与 BERT 预训练的 MLM 策略类似但目的不同——MLM 是让模型预测被 mask 的词来学习语言表示，而我的 masking 纯粹是一种数据增强，目的是提高模型在信息不完整时的鲁棒性，这对 DuReader-robust 数据集尤其重要，因为该数据集本身就测试模型的鲁棒性。

## 三、Dropout 是什么？

### 原理

Dropout 是深度学习中最经典的正则化技术之一。在每次训练的前向传播过程中，**随机地把一部分神经元的输出置为 0**（"丢弃"它们），每次丢弃的神经元是随机选的。

```
训练时（Dropout rate = 0.1，即丢弃 10%）：

输入 → [神经元1: 0.5] [神经元2: ✗丢弃] [神经元3: 0.3] [神经元4: 0.8] [神经元5: ✗丢弃]

推理时（不 Dropout，但输出需缩放）：

输入 → [神经元1: 0.5] [神经元2: 0.2] [神经元3: 0.3] [神经元4: 0.8] [神经元5: 0.1]
```

### 为什么有效？

1. **防止神经元"共适应"（co-adaptation）**：如果某些神经元总是一起工作，模型会变得脆弱。Dropout 强迫每个神经元**独立地**学习有用的特征
2. **隐式集成（ensemble）**：每次 dropout 相当于训练了一个不同的子网络，推理时等效于多个子网络结果的平均

### 训练 vs 推理

| | 训练时 | 推理时 |
|---|---|---|
| Dropout | **开启**，随机丢弃神经元 | **关闭**，使用所有神经元 |
| 原因 | 需要正则化 | 需要完整模型做最优预测 |

> 这就是为什么 PyTorch/PaddlePaddle 中要调用 `model.train()` 和 `model.eval()` 来切换模式。

### 与你实验中 Token Masking 的类比

| | Dropout | 你的 Token Masking |
|---|---|---|
| 丢弃对象 | 神经元的输出 | 输入的 token |
| 丢弃位置 | 模型内部（隐藏层） | 模型外部（输入数据） |
| 丢弃时机 | 每次前向传播时随机选 | 数据预处理时随机选 |
| 核心思想 | **相同：随机遮蔽一部分信息来增强泛化能力** | |

## 四、Softmax 是什么？

### Transformer Attention 公式

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

### Softmax 的定义

**把一组任意的实数，转换成一个"概率分布"**（所有值在 0~1 之间，且加起来等于 1）。

**公式**：

```
softmax(z_i) = e^(z_i) / Σ_j e^(z_j)
```

### 具体计算示例

```
QK^T/√d 计算出的原始注意力分数：[2.0, 1.0, 0.5]

Step 1: 取指数
  e^2.0 = 7.39,  e^1.0 = 2.72,  e^0.5 = 1.65

Step 2: 求和
  总和 = 7.39 + 2.72 + 1.65 = 11.76

Step 3: 归一化
  softmax = [7.39/11.76, 2.72/11.76, 1.65/11.76]
          = [0.63,       0.23,       0.14]
            ↑             ↑           ↑
          最相关        中等相关     不太相关
          
  加起来 = 0.63 + 0.23 + 0.14 = 1.0 ✓ → 合法概率分布
```

### 在 Attention 中各部分含义

| 公式部分 | 含义 |
|---|---|
| `Q` | 查询向量："我在找什么？" |
| `K` | 键向量："我有什么？" |
| `QK^T` | Q 和每个 K 的**相似度原始分数**（点积），值可以是任意实数 |
| `÷ √d_k` | **缩放**：防止点积值太大导致 softmax 梯度消失（值都挤到 0 或 1） |
| `softmax(...)` | 将原始分数变成 **0~1 的注意力权重**，总和为 1 |
| `× V` | 用注意力权重对 V（值向量）做**加权求和**，得到最终输出 |

### 为什么需要除以 √d_k？

```
假设 d_k = 512（维度较高）

不缩放时：QK^T 的值可能很大，如 [50, 2, -3]
softmax([50, 2, -3]) ≈ [1.0, 0.0, 0.0]  → 梯度几乎为零！

缩放后：  [50/√512, 2/√512, -3/√512] ≈ [2.2, 0.09, -0.13]
softmax([2.2, 0.09, -0.13]) ≈ [0.79, 0.10, 0.08]  → 梯度正常
```

### Softmax 的本质

**一句话**：Softmax 是一个**"归一化 + 放大差异"**的函数。
- 分数高的 → 概率被放大（拿到更多注意力）
- 分数低的 → 概率被压缩（几乎被忽略）
- 所有输出加起来 = 1 → 合法的概率分布

### 你实验中 Softmax 的两个用途

| 用途 | 位置 | 作用 |
|---|---|---|
| **Attention 中的 softmax** | Transformer 模型内部 | 计算注意力权重分布 |
| **损失函数中的 softmax** | `softmax_with_cross_entropy` | 将 logit 转为概率，再算交叉熵 loss |

## 五、总结串联

```
                    ┌─ Weight Decay (L2)
                    ├─ Dropout (模型内部)
你的防过拟合手段 ────┤
(正则化)             ├─ Token Masking (输入数据)
                    ├─ FGM 对抗训练
                    └─ Warmup + lr 衰减

Token Masking 发生在 → 微调阶段 → 数据预处理时

Softmax 用于 → Attention 权重计算 + 损失函数
```
