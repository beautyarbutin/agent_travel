## 版本一：适合直接放进简历的项目经历终稿

### 基于 OpenAgents 的多智能体旅游助手系统

**项目角色**：个人项目 / 独立完成  
**技术栈**：OpenAgents、Python、MCP、FAISS、Sentence-Transformers、LLaMA-Factory、LoRA、RAG、高德地图 API、天气 API、Qwen

#### 项目描述

围绕旅游问答与行程规划场景，搭建了一个基于 OpenAgents 的多智能体旅游助手系统，形成了 **Router + 专家 Agent + MCP 工具层 + 结构化记忆 + Hybrid RAG + LoRA 微调 + 离线评测** 的完整链路，覆盖天气查询、景点推荐、路线规划、多轮上下文理解与景点深度攻略检索。

#### 项目亮点（简历推荐写法）

- 基于 OpenAgents 设计并实现 `travel_router + weather_agent + spot_agent + plan_agent` 多智能体协作架构，完成公共频道消息处理、Agent 间 direct message 分发、上下文记忆读写和领域专家解耦。
- 自建 **2 个 MCP Server** 与 **32 个工具/数据/评测脚本**，将天气、景点、路线、记忆等能力以工具协议形式接入 Agent，完成从“自由生成”到“结构化动作调用”的工程落地。
- 构建 **15723 条** 景点知识库，设计 `id/city/district/spot_name/tags/content/duration/budget/rating/source` 半结构化 schema，并实现 **高德 POI + 本地 RAG** 的 hybrid 检索链路。
- 实现基于 `BAAI/bge-small-zh-v1.5 + FAISS IndexFlatIP + BM25-like 词法打分` 的景点检索系统；建库阶段完成向量索引构建，查询阶段完成向量召回、词法补分、手写攻略 bonus 与融合排序。
- 使用 LLaMA-Factory 完成 Router 与 Spot 两套 LoRA 微调，构建 **833 条 Router tool-calling 训练数据**、**8802 条 Spot 问答训练数据**，并建立数据转换、清洗、训练、评测全流程。
- 搭建多套离线评测体系：包含 **34 条 Router 专项评测样本**、**80 条 Spot 内部评测样本**、**150 条 CrossWOZ 景点评测样本**、**198 条 RAG 检索评测样本**，支持工具调用、生成质量、检索命中率和 badcase 归因分析。
- 针对 Router 任务完成专项优化，推动 **sample full-chain success rate 从 20.59% 提升至 85.29%**，**turn exact match 从 48.81% 提升至 94.05%**，**argument exact match 从 19.57% 提升至 89.13%**。
- 连续完成 **7 项以上** 面向真实 badcase 的专项修复，包括事件路由、跨城记忆 schema、城市解析、高德 POI 策略、酒店规划、RAG 数据集扩充、景点消息 fallback 等问题定位与修复。
- 对 RAG 系统完成专项审计与评测，基于 **198 条检索样本**定位精确景点召回、地名解析、chunk 粒度、词法/语义融合等瓶颈，并形成成体系的修复与优化文档。

## 简历里不要夸大的表述

下面这些说法不建议直接写死，容易被问穿：

### 不建议这么写

- “实现了标准 BM25 + rerank 检索系统”
- “落地了成熟的 cross-encoder 重排序模型”
- “实现了严格去中心化多智能体协商”
- “完成了全链路生产级用户隔离记忆系统”

### 更稳妥的写法

- “实现了基于 FAISS + BM25-like 词法补分 + 高德 POI 的 Hybrid RAG”
- “当前未引入独立 cross-encoder rerank，采用融合排序策略”
- “采用 Router + 专家 Agent 的中心化多智能体协作架构”
